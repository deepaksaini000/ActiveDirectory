Forest

Sabse bada AD boundary. Ek company ke paas usually ek hi forest hota hai. Iske andar multiple domains ho sakte hain. Schema (rules) sab domains pe apply hoti hai.

Hindi
Jaise ek desh — usme states hain, states mein cities. Forest = Desh
🏢

Domain

Ek logical boundary jisme users, computers, policies hote hain. Har domain ka apna naam hota hai (corp.local). Domain Controller is sabko manage karta hai.

Hindi
Jaise ek state — apna CM, apne rules, apni police
📁

Organizational Unit (OU)

Domain ke andar containers — jaise folders. IT Department, HR, Finance alag-alag OUs mein. GPO (policies) OU pe apply hoti hain.

Hindi
Jaise office mein alag-alag departments — HR room, IT room
🖥️

Domain Controller (DC)

Sabse important server! Ye AD database (NTDS.dit) hold karta hai. Saari authentication requests yahan aati hain. Har domain mein minimum 2 DCs hone chahiye (backup ke liye).

Hindi
Jaise central police station — sab records yahan hain

Authentication — Login Kaise Hota Hai?
Kerberos aur NTLM — dono samjho
🎡 Kerberos Analogy — Theme Park

Soch ek theme park:
1. Tum ticket counter (KDC) par jaate ho aur apna ID dikhate ho
2. Counter tumhe ek wristband (TGT) deta hai — ye proof hai ki tum legit customer ho
3. Ab kisi bhi ride (resource) ke paas wristband dikhao — seedha andar
4. Har baar counter par nahi jaana — wristband 8 ghante valid hai!

Step 1: AS-REQ

User apna username bhejta hai DC ko. Password se encrypted timestamp bhi bhejta hai. "Main Rahul hoon, mujhe proof karna hai."

Step 2: TGT Mile

DC verify karta hai aur ek Ticket Granting Ticket (TGT) bhejta hai. Ye 8 ghante valid hota hai. Master ticket hai.

Step 3: Service Ticket

Kisi specific resource (file server) ke liye TGT use karo. DC service-specific ticket deta hai. Ab seedha resource access karo.

# Kerberos tickets dekhne ke liye
klist # Current tickets
klist purge # Sab tickets clear karo
klist tgt # TGT specifically dekhna
🔐 NTLM — Purana Tarika (Legacy)

Kerberos se purana protocol. Mostly tab use hota hai jab Kerberos kaam na kare — jaise direct IP se connect karo ya old systems ke saath.

⚠️ NTLM ka Challenge

  • ✗Mutual authentication nahi (server fake ho sakta hai)
  • ✗Pass-the-Hash attacks vulnerable
  • ✗Ticket delegation nahi hoti
  • ✗Performance slower

✅ Kerberos Advantages

  • ✓Mutual authentication (dono side verify)
  • ✓Single Sign-On (SSO) support
  • ✓Ticket delegation — app doosre app ki taraf se act kar sakti hai
  • ✓Faster — DC ko baar baar hit nahi karna
🚨 Security Note

Modern environments mein NTLM disable karna best practice hai. Attackers Pass-the-Hash attack mein NTLM hash steal karke re-use karte hain. Always prefer Kerberos!

Group Policy = Principal ka notice board — ek jagah se rule lagao, sabpe apply ho jaata hai.

🔒

Password minimum 12 characters. Account lockout 5 attempts. Firewall rules force karna. Audit logging enable karna.

Security

User Experience

Proxy settings push karna. Drive mapping (\\server\share → Z: drive). Printers auto-install. VPN configuration deploy.

GPO Application Order — LSDOU Rule

L — Local Policy

Seedha us computer ka local policy. Sabse pehle apply hota hai. Sabse weak — baaki sab isse override kar sakte hain.

S — Site Policy

AD Site (physical location based) ke GPOs. Jaise "Mumbai office" ke liye alag policy.

D — Domain Policy

Poore domain ke liye policy. Password policy yahan set hoti hai. Sabpe apply hoti hai.

OU — Organizational Unit Policy

Specific OU ke liye policy. Sabse last mein apply hoti hai isliye sabse powerful! OU policy domain policy ko override kar sakti hai. Nested OUs mein child OU wins.

💡 Last Policy Wins

Agar Domain Policy kehti hai "USB allowed" aur OU Policy kehti hai "USB blocked" — OU Policy win karega kyunki woh baad mein apply hota hai.

# GPO ke useful PowerShell commands
gpupdate /force # Turant policy refresh karo
gpresult /r # Kaunsi policies apply ho rahe hain
gpresult /h C:\report.html # HTML report banao
Get-GPO -All # Saare GPOs list karo
Get-GPOReport -Name "IT Policy" -ReportType HTML -Path C:\gpo.html

DNS aur Ports — AD ki Backbone
Bina DNS ke AD kaam nahi karta
📞 DNS Analogy

DNS ek phone directory hai. Tum jaante ho dukan ka naam (corp.local), DNS tumhe address (192.168.1.10) deta hai. AD mein bhi sab Domain Controller ka naam DNS se dhundhta hai.

Rule: AD ki DNS fail = AD fail. Simple.

🔍 Key DNS Records in AD

# DC dhundhne ke records
_ldap._tcp.corp.local → DC IP
_kerberos._tcp.corp.local → DC IP
_gc._tcp.corp.local → GC IP

# Manual lookup
nslookup corp.local
nslookup -type=SRV _ldap._tcp.corp.local

⚠️ DNS Troubleshooting

# Login fail ho raha hai? DNS check karo
dcdiag /test:DNS
ipconfig /flushdns
ipconfig /registerdns

# DC reachable hai?
nltest /dsgetdc:corp.local
Test-NetConnection dc1 -Port 389

Azure AD (Entra ID) — Cloud Wala AD
On-prem AD aur cloud AD mein kya fark hai?
☁️ Cloud Analogy

Traditional AD = Ghar mein apna generator — tumhare paas, tumhari responsibility, light jaaye toh tum fix karo.
Azure AD = City ki electricity company — tum sirf bill pay karo, maintenance unki.

Reality: Modern enterprises dono use karte hain — isko Hybrid kehte hain!

🏢 On-Prem AD (Traditional)

  • ✓Full control
  • ✓LDAP, Kerberos, NTLM support
  • ✓Works offline
  • ✗Hardware cost
  • ✗Remote work difficult
  • ✗Mobile devices manage karna mushkil

☁️ Azure AD / Entra ID (Cloud)

  • ✓Remote work perfect
  • ✓Mobile, Mac, Linux support
  • ✓MFA, Conditional Access
  • ✓SaaS apps SSO (Office 365, Salesforce)
  • ✗No Kerberos/NTLM
  • ✗No GPO (Intune use karo)

AD Security — Hackers Kaise Attack Karte Hain
Common attacks aur unka defense
⚠️ Why AD Security Matters

AD ek organization ka brain hai. Agar attacker AD control kar le — toh usne poori company control kar li. 90% ransomware attacks AD ko target karte hain. Isliye AD security = Company security.

⚔️ Common Attacks

  • 💥Pass-the-Hash: NTLM hash steal karo, password bina bhi login karo
  • 💥Kerberoasting: Service account tickets crack karo offline
  • 💥Golden Ticket: krbtgt account compromise karo — lifetime access
  • 💥DCSync: DC ki tarah behave karo, passwords sync karo
  • 💥Password Spray: Common passwords sab accounts par try karo
  • 💥BloodHound: AD ke attack paths map karo automatically

🛡️ Defense Best Practices

  • ✅Tiered Admin Model: Admin accounts alag, daily use accounts alag
  • ✅LAPS: Local admin passwords automatically rotate
  • ✅PAW: Privileged Access Workstation — admin ka dedicated PC
  • ✅Disable NTLM: Where possible
  • ✅Protected Users Group: High-value accounts extra protection
  • ✅Microsoft Defender for Identity: Suspicious activity detect karo

🏛️ AD Tiered Administration Model

Rule
Tier 0 admin kabhi Tier 2 machine par login nahi karega — workstation compromise se DC compromise nahi hoga. Yahi Tiered Model ka point hai!
